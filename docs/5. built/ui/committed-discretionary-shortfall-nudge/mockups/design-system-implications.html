Part B — design-system implications, made concrete

Same code is shipped either way. The difference is what the design system says about it — and therefore what future patterns inherit from it.

D1 · first-class addition

"AttentionStrip" becomes a documented component

We add a new section to design-system.md describing a slim left-panel attention strip that any future left-panel feature can reuse.

What gets added to design-system.md

### `AttentionStrip` A slim attention bar pinned below the left-panel PageHeader. Surfaces a single arithmetic signal that warrants attention without occluding the panel content. Anatomy: - 5px attention dot, body text, optional bold figure inline - Background: attention-bg; border-bottom: attention-border - Hover: tooltip with detail (figures + list) Rules: - Left panel only — never in the right panel (NudgeCard occupies that role) - One AttentionStrip per panel at a time - Arithmetic-only language (Anchor 13)

What this enables tomorrow

A future feature wants to flag "ISA allowance ending in 14 days" on the Surplus left panel.
→ Reuse AttentionStrip. No new design discussion needed.
A designer wants "linked accounts last synced 3h ago" on the Forecast page.
→ Reuse AttentionStrip. The pattern is sanctioned.

Cost

  • One extra design-doc update this session.
  • Slight risk of the pattern being misused later (anything amber gets jammed into a strip).
D2 · scoped exception

"ShortfallStrip" is purpose-built for this nudge

The strip ships, but the design system stays silent on it as a general pattern. The spec notes it's a one-off for cashflow shortfalls.

What gets added to the spec

> Implementation note The shortfall surface introduces a new left-panel attention strip scoped to this feature only. Do not generalise into a reusable component without a follow-up design pass. design-system.md is unchanged.

What this means tomorrow

A future feature wants to flag "ISA allowance ending in 14 days" on the Surplus left panel.
→ Cannot reuse the shortfall strip. Run a fresh design pass first.
A designer copies the shortfall strip CSS into a Forecast warning.
→ Code-review feedback: this is undocumented; needs a design decision.

Cost

  • Each future left-panel attention need triggers another design conversation.
  • Risk: people copy the shortfall strip anyway because no documented alternative exists, and divergence creeps in.

Both options ship the same UI for this feature. The choice is whether the pattern is reusable next time without re-deciding.
