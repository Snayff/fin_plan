Settings — Current State Audit

Problems I've identified before designing the enhanced version

Bug
Nav highlight misalignment
  • Active item uses bg-accent text-accent-foreground — a full background fill rather than the design system's indicator pattern
  • Should be: bg-page-accent/14 border-l-2 border-page-accent rounded-r-sm
  • The click-to-scroll and scroll-spy are also slightly out of sync — clicking an item doesn't reliably set the active highlight
Bug
Household dropdown off-screen
  • The switcher dropdown uses absolute top-full left-0 — positioned relative to the trigger element
  • Since the trigger is near the right edge of the nav bar, the dropdown can overflow the viewport on the right
  • Fix: use right-0 instead of left-0 to anchor to the right edge, or add a max-h + overflow-y-auto for long lists
Design System
Left panel doesn't match design system
  • Width is 192px (w-48) — design system specifies 360px for the left panel
  • No PageHeader component — using inline <p class="label-section"> instead
  • No left panel scroll structure (flex flex-col h-full with flex-1 overflow-y-auto)
UX
Excessive whitespace in right panel
  • Content is constrained to max-w-2xl (672px) in a potentially very wide right panel
  • Settings sections render as a scrollable long page — sections feel disconnected
  • No visual hierarchy between sections — they blend together
Architecture
Household settings in disparate places
  • Household switching is in the top nav (HouseholdSwitcher)
  • Household management (rename, invite, leave) is buried in Settings → Household
  • Creating a new household is in the top nav dropdown
  • These feel fragmented — user has to hunt for household-related actions
UX
Inconsistent save pattern
  • Display: auto-saves on checkbox toggle ✓
  • Profile: requires explicit Save button
  • Staleness thresholds: requires explicit Save button
  • Surplus benchmark: requires explicit Save button
  • Inconsistency creates cognitive friction — user doesn't know what will be lost on navigate
