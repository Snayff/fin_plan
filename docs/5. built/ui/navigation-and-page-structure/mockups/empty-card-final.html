Empty State Copy — Pattern & Examples

Defining the language rule so every empty state reads consistently.

Empty state copy pattern
Header: Add your [context]
Body:  [examples of what belongs here]
  • Header — starts with an action verb. Names the context (subcategory or page). Short — one line max.
  • Body — provides 2–4 concrete examples separated by commas. Helps the user understand what goes in this subcategory. No full sentences — just a list of nouns.
  • Tone — direct and helpful, never instructional. "Add your housing costs" not "You should add your housing costs". No exclamation marks.
  • Button — always "+ Add item" for subcategories. "Get started" for Overview (launches build wizard).
  • No "first" — avoid "Add your first item". The word "first" implies a sequence that may not exist. Just "Add your [context]".
