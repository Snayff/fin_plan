Fields we have per tier. Design principle 3 says "precision without density" — spacious, scannable, not spreadsheet-like.

A — Full density (6 columns). Every tier-specific field is an inline column. Fast tab-through for power users; every data point visible without drilling in. Risks feeling spreadsheet-y, which the design principles warn against.

B — Lean (4 columns). Name includes the type chip; owner and incomeType live in the row's expand-to-edit panel. Calmer scan; one extra click to edit owner/type. Matches "earn every pixel" and "precision without density".

C — Grouped by subcategory. Rows cluster under subcategory labels (Housing, Insurance…). Due column is conditional — shown only on yearly/one-off rows. "/month" column is the waterfall-relevant figure (yearly ÷ 12) — this is the honest cascade number.

Subcategory as cell; rows freely sortable/filterable.

Pick one density (A or B) and one layout (C or D). My lean: B + C — lean row + grouped by subcategory reinforces the waterfall's hierarchical buckets and stays calm.
