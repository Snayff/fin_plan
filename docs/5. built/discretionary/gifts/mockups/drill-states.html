Colour coding — design-anchor check

I'm using subtle purple tint for "bought" counts, muted text-tertiary for "planned" counts. This avoids red/green (Anchor #10 — non-judgemental) and stays calm. Flag: the purple is close to the discretionary tier colour — tier colours are meant to be semantically protected (§1.2 of design-system). Two options:

  • a) Keep as-is — the purple tint is subtle enough that it reads as "count that matters" rather than "this is a tier colour". Document as a scoped exception.
  • b) Use weight/opacity only — bought is text-primary with weight 500, planned is text-tertiary with weight 400. No colour hue difference. Purely monochromatic distinction.
  • c) Use a checkmark icon on bought, no colour — add a small ✓ symbol next to the bought count. Icon conveys state, text stays monochromatic.

My recommendation: b) — safest with the anchors, still visually distinct. Let me know and I'll update.
